## What is Zypace?

Zypace is a personalized running training platform that generates science-based training plans using AI. It connects with your Strava account to monitor your workouts, adapts the plan in real time based on your fatigue, recovery, and race calendar, and gives you day-by-day guidance — from easy runs to strength sessions to trail-specific climbing blocks.

Unlike generic plans, Zypace builds your program around **who you are as a runner**: your experience level, injury history, available days, target race, terrain, and elevation profile.

## Key Features

### AI-Generated Training Plans
- First mesocycle generated by GPT-4o based on full runner profile
- Subsequent mesocycles generated automatically as each block completes
- Scope-aware regeneration: update only running sessions, only strength, or both
- Polarized, Norwegian double-threshold, and classic periodization methodologies
- Multi-race calendar: tapers and load distribution across A/B/C priority races

### Trail Running Support
- `elevation_gain_m` (D+) field on races — the AI classifies the plan differently for suave, medio, montaña alta, and ultra extremo categories
- `subida` workout type: sessions expressed in `Xmin/YD+` format with RPE-based intensity (pace is irrelevant on mountain terrain)
- Power hiking blocks automatically included for races >3000D+ or >50 km
- Downhill technique sessions in the specific phase
- Trail-specific strength work: eccentric quad loading, peroneals, hip thrust for ascent power

### Smart Training Calendar
- Weekly grid with 8 workout types: suave, largo, series, umbral, tempo, subida, fuerza, descanso
- Today/Tomorrow quick-view banners with completion toggles
- Progress bars per week and per mesocycle
- D+ shown alongside km/min for trail sessions
- Auto-scroll to current week on load

### Strava Integration
- OAuth 2.0 — tokens stored server-side, never exposed to the frontend
- Real-time webhook sync: activities automatically complete matched workouts
- Saves heart rate, cadence, elevation gain, suffer score, and power per activity
- Full history import on first sync; incremental updates thereafter

### Fatigue & Readiness System
- Fatigue index computed from RPE, feelings, and recent adherence
- Weekly check-in (readiness + life context) informs the next mesocycle load
- Mesocycle history snapshots stored with adherence %, total km, and avg RPE
- Load adjustments: rest weeks, lighter weeks, or progressive overload based on signals

### Strength Training
- Running-specific sessions: eccentric chain, max neuromuscular strength, plyometrics
- Three complementary sessions per week (A/B/C structure — no repeated movements)
- Phase-aware progression: base → development → specific → taper
- Injury-aware: adapts exercises around chronic and recent injury areas

### Subscriptions & Monetization
- 30-day free trial via Stripe Checkout
- Discount codes: percentage, fixed amount, time-limited durations
- Self-serve billing management via Stripe Customer Portal
- Admin panel: manage users, exemptions, discount codes, incidents, NPS responses

### Firestore Data Model

```
users/{uid}
  ├── races/{raceId}              # Goals: date, distance, terrain, D+, priority (A/B/C)
  ├── training_plans/default      # Single plan per user: primary_race_id, config, mesocycle state
  ├── workouts/{workoutId}        # Daily sessions: description, type, elevation_gain_m, RPE…
  ├── strava_activities/{actId}   # Raw Strava activity data
  ├── weekly_reviews/{reviewId}   # Athlete readiness check-ins
  └── mesocycle_history/{id}      # Completed block snapshots (adherence, km, RPE)

strava_athlete_index/{athleteId}  # athlete_id → uid mapping for webhook routing
discount_codes/{code}
invites/{inviteId}
```

## Getting Started

### Prerequisites

- Node.js 22+
- Firebase CLI (`npm install -g firebase-tools`)
- Firebase project with Firestore, Auth, Functions, and Hosting enabled
- Strava API application (for OAuth + webhooks)
- OpenAI API key
- Stripe account

### Frontend Setup

```bash
# Clone the repo
git clone https://github.com/Diego-DPL/zypace.git
cd zypace

# Install dependencies
npm install

# Configure environment
cp .env.example .env.local
# Fill in your Firebase config, Strava Client ID, and GA4 ID

# Start dev server
npm run dev
```

### Environment Variables

```env
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
VITE_STRAVA_CLIENT_ID=
VITE_GA_MEASUREMENT_ID=
```

### Cloud Functions Setup

```bash
cd functions
npm install

# Set Firebase secrets (never committed to source control)
firebase functions:secrets:set OPENAI_API_KEY
firebase functions:secrets:set OPENAI_MODEL          # e.g. gpt-4o
firebase functions:secrets:set STRAVA_CLIENT_SECRET
firebase functions:secrets:set STRIPE_SECRET_KEY
firebase functions:secrets:set STRIPE_WEBHOOK_SECRET
firebase functions:secrets:set RESEND_API_KEY
```

### Deploy

```bash
# Deploy everything
firebase deploy

# Frontend only (fast iteration)
firebase deploy --only hosting

# Functions only
firebase deploy --only functions
```

## Training Plan Intelligence

Plans are structured around **mesocycles** (4–6 week blocks). Each generation call provides the AI with:

| Signal | Data |
|---|---|
| Runner profile | Experience level, age, weekly volume, longest run, session limits, injury history |
| Race context | Name, date, distance, D+ elevation, terrain (road/trail/mixed/track), priority |
| Multi-race calendar | All upcoming races with A/B/C priorities for taper planning |
| Fatigue signals | RPE logs, feelings (great/tired/very tired), recent adherence percentage |
| Weekly check-in | Readiness score and life context for the upcoming training block |
| Pace zones | Calibrated from target time or stored profile data |

The structured prompt enforces **periodization phases** (base → development → specific → taper), **day-by-day schedule constraints**, **methodology rules** (polarized 80/20, Norwegian double-threshold, or classic), and **trail-specific session types** based on race elevation profile.
